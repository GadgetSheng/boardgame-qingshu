# 情书 (Love Letter) - 单机桌游 2019版

## 1. 项目概述

- **项目名称**: 情书 (Love Letter) 2019新版
- **类型**: 单机可玩的卡牌桌游
- **核心功能**: 支持与AI对战的2-6人情书卡牌游戏
- **目标用户**: 想要线下或单人练习情书规则的玩家

## 2. 游戏规则

### 2.1 游戏配置
- 玩家数: 2-6人 (含1-5个AI)
- 卡牌组: 21张卡
- 胜利条件: 先到4分的玩家获胜；游戏结束时手牌点数最大者获胜；若弹劾公主则立即失败

### 2.2 卡牌分布
| 卡牌 | 点数 | 张数 | 效果 |
|------|------|------|------|
| 公主 (Princess) | 9 | 1 | 弃掉即出局 |
| 伯爵夫人 (Countess) | 8 | 1 | 与国王/王子同在时必须打出 |
| 国王 (King) | 7 | 1 | 与一名玩家交换手牌 |
| 大臣 (Chancellor) | 6 | 2 | 抽2选1，其余放回牌库底 |
| 王子 (Prince) | 5 | 2 | 选择一人强制弃手牌并重抽 |
| 侍女 (Handmaid) | 4 | 2 | 免疫一轮指向性攻击 |
| 男爵 (Baron) | 3 | 2 | 与一人秘密比大小，小者出局 |
| 神父 (Priest) | 2 | 2 | 秘密看一人手牌 |
| 守卫 (Guard) | 1 | 6 | 猜一人手牌（不能猜守卫），猜对目标出局 |
| 间谍 (Spy) | 0 | 2 | 独家打出且存活至局末，额外奖励1分 |

### 2.3 游戏流程
1. 开局移牌: 从牌堆顶移除1张置于弃牌区
2. 发牌: 每位玩家1张手牌，剩余牌堆放中央
3. 回合: 抽牌 → 出一张牌 → 结算效果
4. 存活条件: 手牌被猜中或弹劾公主则淘汰
5. 结算: 剩1人 / 牌堆清空 → 点数最大者胜 + 间谍奖励
6. 多轮制: 率先获得4分的玩家获胜

### 2.4 AI行为
- 简单AI: 随机出牌
- 普通AI: 优先保留高点数，会使用大臣滤牌

### 2.5 特殊规则
- 伯爵夫人: 拥有王子或国王时必须出伯爵夫人
- 间谍奖励: 唯一在弃牌堆中有间谍的玩家额外+1分
- 大臣选择: 出大臣后需从3张牌中选择1张保留

## 3. 视觉规范

### 3.1 主题风格
- 中世纪宫廷风格
- 配色: 深红/金色/象牙白
- 字体:衬线体标题，无衬线正文

### 3.2 牌面设计
- 牌背: 深红色 + 金色花纹
- 正面: 象牙白底 + 角色图标 + 卡名 + 点数
- 尺寸: 120x168px (标准扑克牌比例)

### 3.3 布局
- 顶部: 牌堆 + 弃牌区 + 积分显示
- 中间: 其他玩家区域
- 底部: 玩家手牌 + 操作按钮
- 消息区: 回合提示 + 历史记录

### 3.4 动画
- 出牌: 滑入弃牌区 300ms ease-out
- 抽牌: 从牌堆飞入手牌 200ms
- 淘汰: 淡出 + 划线 500ms

## 4. 技术方案

- **框架**: React 18 + Vite
- **样式**: CSS Modules
- **状态管理**: React useState/useReducer
- **构建**: 静默桌面应用 (可选)
